### Requirements

- [NextJS 15](https://nextjs.org) or later
- [React 19](https://react.dev) or later
- [Tailwind Css 3.4](https://tailwindcss.com/) or later

### Starter Template

Manual setups can be time-consuming, so I created a [nextjs-starter-manual](https://github.com/mijn-ui/nextjs-starter-manual) template to help you get started quickly you can either use this or follow the steps [below](#setting-up-manually) to integrate Mijn-UI components manually.

If you have an existing project and need to integrate Mijn-UI components manually, simply copy the `mijn-ui` folder from the [nextjs-starter-manual](https://github.com/mijn-ui/nextjs-starter-manual) and install the necessary dependencies used by this folder.

---

### Setting Up Manually

<Steps>
  <Step>
**Install Next.js and Required Dependencies**

```bash title="bash"
  npx create-next-app@latest
```

Since we are in the beta stage, the following options work pretty well for installation. Please ensure to select these options:

```bash title="bash"
  ...
  ✔ Would you like to use TypeScript? Yes
  ✔ Would you like to use ESLint?  Yes
  ✔ Would you like to use Tailwind CSS? Yes
  ✔ Would you like your code inside a `src/` directory? Yes
  ✔ Would you like to use App Router? (recommended) Yes
  ✔ Would you like to use Turbopack for next dev? Yes
  ✔ Would you like to customize the import alias (@/* by default)? No
```

After setting up the Next.js project, install these dependencies used by Mijn-UI components:

```package-install
react-icons class-variance-authority tailwindcss-animate clsx tailwind-merge
```

</Step>
<Step>

**Configure TypeScript Path Aliases**

Since this project is still in a beta phase, and extensive testing is ongoing, we recommend configuring TypeScript with path aliases for better project organization and readability. Please use the following setup in your tsconfig.json:

```json doc-gen:file
{
  "file": "./examples/react/_configs/tsconfig-example.json",
  "codeblock": {
    "lang": "json",
    "meta": "title=\"tsconfig.json\""
  }
}
```
**Recommended Folder Structure**

I recommend using this folder structure for manual integration, similar to the packages folder structure. This way, you won't need to update any component paths, as the source codes are from the mijn-ui packages folder.

<Files>
  <Folder name="public">
    <File name="..." />
  </Folder>
  <Folder name="src">
    <Folder name="app">
      <File name="page.tsx" />
      <File name="..." />
    </Folder>
  </Folder>
  <Folder defaultOpen name="mijn-ui">
    <Folder name="components">
      <File name="react-accordion.tsx" />
      <File name="react-button.tsx" />
      <File name="..." />
    </Folder>
    <Folder name="utilities">
      <Folder name="context">
        <File name="index.ts" />
      </Folder>
      <Folder name="shared">
        <File name="index.ts" />
      </Folder>
      <File name="..." />
    </Folder>
    <Folder name="hooks">
      <File name="use-controlled-state.ts" />
      <File name="..." />
    </Folder>
  </Folder>
</Files>

</Step>

<Step>

**Add Themes**

These are the default themes provided by mijn-ui. If you would like to create your own theme, learn more about themes [here](/docs/theming).

```json doc-gen:file
{
  "file": "./examples/react/_configs/global.css",
  "codeblock": {
    "lang": "css",
    "meta": "title=\"global.css\""
  }
}
```

</Step>

<Step>
**Configure tailwind**

Since our components are stored in a separate `mijn-ui` folder, we need to include this folder in the Tailwind CSS content array. This ensures Tailwind scans these files to generate the required CSS classes:

```json doc-gen:file
{
  "file": "./examples/react/_configs/tw-config-manual.js",
  "codeblock": {
    "lang": "js",
    "meta": "title=\"tailwind-config.js\""
  }
}
```

</Step>

    <Step>
